## 🧭 한 줄 요약

메이플스토리에서 경제 왜곡을 유발하는 **'주차 유저'를 클러스터링으로 검출**하고, 이들의 행동 특징을 데이터 기반으로 분석하여 디렉터가 직면한 타겟팅 문제에 대한 **Data-Driven 대안**을 제시한다.

---

## 🗂️ 문제 배경

### 디렉터 발언 (라이브 방송 직접 언급)

> *"성장 수준을 고정하고 시장에 공급하는 경우, 소위 주차 유저들의 메소 생산량만 줄이려고 해봤지만 그분들만 타겟팅해서 줄이는 게 현재 시점에서는 어려웠다."*
> 

> — 메이플스토리 디렉터, 라이브 방송 중
> 

### 주차 유저란?

- **정의**: 캐릭터의 성장(레벨업, 전투력 향상)을 의도적으로 멈추고, 특정 성장 구간에 고정된 채 **주간 보스 레이드를 반복**하여 **메소(게임 내 화폐)를 대량 생산**하는 유저. 사냥보다 보스 드롭 수익이 주된 수익원임
- **문제**: 이들의 대규모 메소 공급은 인플레이션을 유발하며, 게임 경제 밸런스를 교란함
- **현재 한계**: 일반 유저와 행동 패턴이 외관상 유사해 **규칙 기반 타겟팅이 어려운 상황**

---

## 🔬 연구 목적

1. Nexon OpenAPI 데이터를 활용하여 **주차 유저 집단을 클러스터링으로 검출**
2. 클러스터별 통계적 특징을 분석하여 **주차 유저가 집중되는 레벨 구간·직업군을 특정**
3. 분석 결과를 토대로 **낮은 오분류율을 유지하는 Data-Driven 타겟팅 Rule** 제안

---

## 💡 가설 3개

### 가설 1 — 클러스터링으로 주차 유저 집단 분리 가능성

> **"주차 유저는 일반 유저와 플레이 패턴 feature 상에서 구별되는 군집을 형성할 것이다."**
> 
- **검증 방법**: K-Means 또는 DBSCAN 클러스터링 (Elbow Method / Silhouette Score로 최적 k 선정)
- **주요 Feature 후보**:
    - 지난 2년간 레벨 변화량 (Δlevel)
    - 지난 2년간 전투력 변화량 (Δ전투력)
    - 지난 2년간 유니온 레벨 변화량 (Δunion)
    - 연무장(무릉도장) 최고 층수 변화
    - 심볼 성장 수준 (아케인/사크라멘트)
- **기대 결과**: 성장 지표(레벨·전투력·유니온) 변화가 거의 없는 군집이 별도로 형성됨

---

### 가설 2 — 주차 유저는 특정 레벨 구간·직업군에 불균형하게 집중 분포한다

> **"주차 유저 군집은 전체 유저 대비 특정 레벨 구간 및 직업군에서 통계적으로 유의미하게 높은 비율로 나타날 것이다."**
> 
- **검증 방법**: 카이제곱 검정(Chi-Square Test)으로 레벨 구간별·직업별 주차 유저 비율 분포 검정
- **검정 대상**:
    - 레벨 구간별 (예: 240~260, 260~280, 280~300 등) 주차 유저 비율
    - 직업군별 주차 유저 비율 (보스 효율이 높은 직업군 집중 여부)
- **유의수준**: α = 0.05
- **기대 결과**: 특정 레벨 구간 및 직업군에서 주차 유저 비율이 유의미하게 높음 (p < 0.05)
- **⚡ Action Required**: 주차 유저가 집중된 레벨 구간과 직업군을 데이터로 특정함으로써, 넥슨이 **해당 구간의 보스 드롭률·메소 수급량을 선별 조정**할 수 있는 근거 제공. 전체 유저가 아닌 특정 구간에만 정책을 적용하는 것이 가능함을 시사

---

### 가설 3 — 도출한 Rule-Based 기준은 낮은 오분류율로 주차 유저를 식별할 수 있다

> **"Feature Importance 기반으로 도출한 Rule-Based 타겟팅 기준은, 일반 유저를 주차 유저로 잘못 분류하는 비율(False Positive Rate)을 허용 가능한 수준으로 유지하면서 주차 유저를 식별할 수 있다."**
> 
- **검증 방법**:
    - Random Forest 또는 XGBoost로 Feature Importance 산출 후 핵심 feature 기반 Rule 도출
    - Rule의 분류 성능을 Precision / Recall / F1-Score / False Positive Rate로 평가
    - ROC-AUC 곡선으로 Threshold별 Trade-off 시각화
- **기대 결과**: 예) "레벨 변화량 < N AND 전투력 변화량 < M" Rule이 FPR X% 이하를 유지하면서 주차 유저 탐지 가능함을 수치로 증명
- **⚡ Action Required**: 디렉터가 "그분들만 타겟팅해서 줄이는 게 어려웠다"고 언급한 문제에 대해, **일반 유저 피해를 최소화하면서 적용 가능한 Rule을 수치 근거와 함께 제시**. 넥슨 정책팀이 실제 시스템에 도입할 수 있는 데이터 기반 타겟팅 기준을 제공함

### ❌ 제공되지 않는 데이터

- **보스 클리어 이력**: 보스별 클리어 기록 없음
- **접속 시간 / 접속 주기**: 로그인 로그 제공 안 함
- **사냥터 체류 패턴**: 어떤 맵에서 얼마나 사냥했는지 불가
- **메소 직접 추적**: 메소 획득/소비 이력 없음
- **큐브/스타포스 이력**: 계정주 본인만 조회 가능 → 공개 수집 불가 (제외)

### 💡 Feature 선정: 가설별 사용 데이터

#### 가설 1 (Clustering) Feature 후보

| Feature | 수집 방법 | 주차 유저 예상 패턴 |
| --- | --- | --- |
| 레벨 변화량 (Δlevel) | `basic` API, date별 추적 | 거의 0에 수렴 |
| 전투력 변화량 (Δ전투력) | `stat` API, date별 추적 | 매우 단조로움 (장비 강화 없이 고정) |
| 유니온 레벨 변화량 (Δunion) | `union` API, date별 추적 | 매우 단조로움 |
| 연무장(무릉도장) 최고층 변화량 | `character/dojang` | 일정 층 유지, 변화 적음 |
| 심볼 성장 수준 | `character/symbol-equipment` | 성장 정체로 거의 고정 |

#### 가설 2 (카이제곱 검정) 분석 대상

| 변수 | 수집 방법 |
| --- | --- |
| 레벨 구간별 주차 유저 비율 | `basic` 레벨 + 클러스터 레이블 교차 집계 |
| 직업군별 주차 유저 비율 | `basic` 직업 + 클러스터 레이블 교차 집계 |

#### 가설 3 (분류 성능 평가) 분석 대상

| 평가 지표 | 의미 |
| --- | --- |
| Precision | 주차 유저로 분류된 것 중 실제 주차 유저 비율 |
| Recall | 실제 주차 유저 중 Rule이 잡아낸 비율 |
| False Positive Rate | 일반 유저를 잘못 주차 유저로 분류한 비율 (핵심 지표) |
| ROC-AUC | Threshold별 Trade-off 종합 평가 |

### 수집 전략

- 랭킹 API (`ranking/overall`)로 **다양한 레벨 구간 캐릭터 샘플링** (편향 방지)
- 수집 주기: `date` 파라미터로 월 1회 스냅샷
- 예상 샘플 수: **성장구간별 캐릭터 200~500개** (rate limit 고려)
- 수집 기간: 2023년 5월 ~ 2025년 5월 (2년)
